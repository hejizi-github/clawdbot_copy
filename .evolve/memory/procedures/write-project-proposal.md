**适用场景**: 基于调研/分析产出一个新项目提案（选什么做 + 技术设计 + 实施计划）

**步骤**:
1. 回顾前序调研报告，提取 impact × feasibility 矩阵中的候选方向
2. 选定方向后，做竞品分析——对每个竞品的版本号和关键功能通过 PyPI/GitHub releases 交叉验证
3. 确定项目名称后，立即检查包注册表（PyPI/npm/crates.io）名称可用性
4. 撰写技术架构设计，代码示例中的技术选型必须与正文描述一致（如选 Pydantic 就不能出现 @dataclass）
5. 制定 MVP 分阶段计划，明确每个 session 的交付物
6. 自审一遍：日期、数字、列表完整性、术语一致性

**注意事项**:
- 名称冲突是最容易遗漏也最浪费时间的问题，务必在步骤 3 卡住
- 竞品描述不准确会影响"为什么不用现有工具"的说服力，宁可少写也不要写未验证的信息
