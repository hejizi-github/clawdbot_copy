# Session 20260417-055511 Plan — Calibration Threshold + Small Fixes

## Goal
Add `--threshold` pass/fail gate to the `calibrate` CLI command (exit 0/1 based on Spearman ρ ≥ threshold), enabling CI-ready calibration quality checks. Fix two minor issues from Session 10 review.

## Rationale
The `eval`, `judge`, and `compare` commands all have threshold/pass-fail CI integration. The `calibrate` command is the only one missing this — it always exits 0. The proposal targets ≥0.80 Spearman as the core quality metric, but there's no way to enforce it programmatically.

## Changes

### Feature: Calibration threshold (Priority 1)
1. **`trajeval/trajeval/cli.py`** — Add `--threshold` option to `calibrate` command (default 0.80). Add pass/fail logic: `overall_spearman_rho >= threshold`. Return exit code 1 on fail. Include `passed` and `threshold` in JSON output.
2. **`trajeval/trajeval/calibration.py`** — Add `passed` field to `CalibrationResult` (computed after correlation, based on threshold). Or: keep threshold logic in CLI only (like `judge` does) — this is simpler and consistent with the existing pattern.
3. **`trajeval/tests/test_cli.py`** — Add tests for calibrate threshold: pass case, fail case, JSON output includes passed/threshold.
4. **`trajeval/tests/test_calibration.py`** — Add tests: moderate correlation (between 0.5-0.8) correctly evaluated against different thresholds.

### Fix: NamedTemporaryFile cleanup (Priority 2 from review)
5. **`trajeval/tests/test_integration.py`** — Replace `NamedTemporaryFile(delete=False)` with `tmp_path` fixture in `test_empty_trace_cli_eval`.

### Fix: Mock verification (Priority 2 from review)
6. **`trajeval/tests/test_integration.py`** — Add `assert high_client.call_count > 0` (or equivalent) in `test_judge_cli_with_fake_client_exit_codes` to verify the fake client was actually invoked.
