# Plan: Add OpenTelemetry (OTLP) JSON Trace Ingestion

## Goal
Add OTLP JSON trace format support to trajeval, making it interoperable with any OpenTelemetry-exporting agent framework.

## Why This
- The project's mission is "framework-agnostic" evaluation, but currently only supports Clawdbot JSONL and a custom JSON format
- OTLP is the industry standard for distributed tracing (CNCF graduated project)
- Most modern agent frameworks can export OTLP traces (LangChain, CrewAI, etc.)
- This is a core feature addition, not test maintenance — aligns with review feedback

## Changes
1. **`trajeval/ingester.py`** — Add `ingest_otlp_json()` function that:
   - Parses OTLP JSON export format (`resourceSpans` → `scopeSpans` → `spans`)
   - Converts spans to TraceSteps (maps span kind/attributes to step types)
   - Computes duration from `startTimeUnixNano`/`endTimeUnixNano`
   - Handles the OTLP attribute array format (`[{"key": "k", "value": {"stringValue": "v"}}]`)

2. **`trajeval/cli.py`** — Add `otlp` to `--input-format` choices, wire up auto-detection for `.otlp.json` files

3. **`trajeval/tests/`** — Add test fixtures and tests for:
   - Basic OTLP ingestion (happy path)
   - Nested spans → flat step list
   - Missing/optional fields
   - Error spans (status.code == 2)
   - CLI integration with `--input-format otlp`
