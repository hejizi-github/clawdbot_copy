# Session Plan: Fix Weak Assertions + GitHub Actions CI

## 目标
修复 e2e 测试中 2 个弱断言 + 为 trajeval 搭建 GitHub Actions CI 工作流

## 方案

### Part 1: 修复弱断言 (test_e2e.py)

1. **Line 309** `test_tolerance_changes_regression_detection`:
   - 当前: `assert data_tight["has_regression"] is True or data_loose["has_regression"] is False` (析取式，一个满足即通过)
   - 实测: tight→True, loose→False，两个都成立
   - 修复: 拆成两个独立 assert，各自验证

2. **Line 332** `test_recovery_window_affects_score`:
   - 当前: `assert short_recovery["score"] != long_recovery["score"] or True` (no-op)
   - 实测: 两个 score 相同 (0.3333)，recovery_window 不改变 score 但确实 flow-through 到 details
   - 修复: 改为断言 recovery_window 值出现在 details 中且与传入参数匹配——验证参数 flow-through 而非断言不成立的 score 差异
