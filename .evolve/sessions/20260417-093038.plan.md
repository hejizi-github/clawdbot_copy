# Session Plan: Clawdbot JSONL Transcript Ingestion

## Goal
Add Clawdbot JSONL transcript ingestion to trajeval, closing the loop between Phase 1 (clawdbot architecture study) and Phase 3 (building). This is an explicit post-MVP target from project-proposal.md Section 7.

## Rationale
This is the highest-value single change because:
1. It demonstrates real-world applicability (not just custom JSON)
2. Connects the three strategy phases into a coherent narrative
3. Enables evaluating actual Clawdbot agent sessions
4. The three ACTIVE goals are already complete — this advances Phase 3 construction

## Plan

### Files to modify
- `trajeval/trajeval/ingester.py` — add `ingest_clawdbot_jsonl()` function
- `trajeval/trajeval/cli.py` — add `--format clawdbot` option to eval command
- `trajeval/tests/test_ingester.py` — add tests for Clawdbot JSONL parsing

### Mapping: Clawdbot JSONL → AgentTrace

| Clawdbot Entry | TraceStep |
|---------------|-----------|
| Session header (type="session") | AgentTrace.metadata (session_id, version, cwd) |
| Assistant message with text content | type="llm_call", name=model, tokens from usage |
| Assistant message with toolCall content | type="tool_call", name=tool_name, input=arguments |
| toolResult message | Merged into preceding tool_call's output |
| bashExecution message | type="tool_call", name="bash", input={command}, output={stdout, exitCode} |
| Compaction entry | Skipped (internal bookkeeping) |
| User message | Used as context for following assistant step's input |

### Duration calculation
- Timestamps on each entry → compute duration_ms as diff between consecutive steps

### Token mapping
- Clawdbot: `usage.input` / `usage.output` → TokenUsage: `prompt` / `completion`
- Clawdbot: `usage.cost.total` → metadata
