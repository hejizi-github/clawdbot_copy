# Session 20260417-070605 Plan — compare --details flag

## 目标
为 `trajeval compare` 的 table 输出添加 `--details` flag，展示每个指标的 baseline/current 诊断详情，保持与 `eval --details` 的 UX 对称性。

## 方案

### 1. CLI 修改 (`trajeval/trajeval/cli.py`)
- 在 `compare` command 添加 `--details` Click option (is_flag=True, default=False)
- 传递 `show_details` 参数到 `_print_comparison`
- 修改 `_print_comparison` 添加 "Baseline Details" 和 "Current Details" 两列
- 用已有的 `_format_details_compact` 渲染 details
- `--format json` 和 `--format markdown` 时忽略 `--details`（json 已有完整 details，markdown 已有 collapsible sections）

### 2. 测试 (`trajeval/tests/test_cli.py`)
- `test_details_flag_shows_details_columns`: 验证 table 输出包含 "Baseline Details" 和 "Current Details"
- `test_no_details_flag_hides_columns`: 验证默认无 details 列
- `test_details_flag_shows_metric_info`: 验证 details 内容可见
- `test_details_flag_with_json_format_ignored`: 验证 --format json 时 --details 不影响输出
- `test_details_flag_with_markdown_format_ignored`: 验证 --format markdown 时 --details 不影响输出

## 验证
- `python -m pytest trajeval/tests/ -q` — 所有测试通过
- 新增测试 ~5 个
- 手动检查：`python -m trajeval compare <trace> <trace> --details` 输出格式正确

## Checklist (对照已有模式)
- [ ] CLI flag 风格与 eval --details 一致（is_flag=True）
- [ ] _format_details_compact 复用，不重复实现
- [ ] Overall 行有空字符串占位
- [ ] json/markdown 格式不受影响
