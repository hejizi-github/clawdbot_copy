## Goal
Fix loop detection n-gram double-counting penalty and fix `format_judge_ci` unsafe `passed=True` default.

## Problem Analysis

### Loop detection double-counting
With default `ngram_sizes=[2, 3]`, a sequence like `A B A B A B` (6 steps) gets:
- 2-gram `(A,B)` x3, `(B,A)` x2 → repeated_steps = 2*(3-1) + 2*(2-1) = 6
- 3-gram `(A,B,A)` x2, `(B,A,B)` x2 → repeated_steps = 3*(2-1) + 3*(2-1) = 6
- Total: 12 repeated steps for a 6-step trace → penalty = min(12/6, 0.9) = 0.9 → score = 0.1

The actual wasted steps are ~4 (2 extra repetitions of the cycle), not 12. Overlapping n-gram sizes double-count the same redundancy.

**Fix**: After collecting all detected loops across n-gram sizes, deduplicate by removing shorter patterns that are subsequences of longer detected patterns. Then recalculate `total_repeated_steps` from only the non-subsumed patterns.

### `format_judge_ci` `passed` default
Current: `passed: bool = True` — unsafe default for a public API, could mask failures.
Fix: `passed: bool | None = None`, auto-compute from `result.overall_score >= threshold` when None.
