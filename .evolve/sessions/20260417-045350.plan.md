# Session 20260417-045350 Plan — Compare + Regression Detection (Phase 3, Session 4)

## Goal

Implement `trajeval compare <baseline> <current>` command for regression detection — the feature that makes trajeval actionable in CI pipelines.

## Why This Is the Highest-Impact Change

1. Per the proposal, Session 4 is "Comparison + regression detection" — the natural next step
2. The review explicitly ranked this as Priority 1: "letting the tool be truly useful in CI"
3. Without `compare`, users can only evaluate single traces — they can't detect regressions between versions

## Plan

### Fix 1: `judge --threshold` (small, from review Priority 2)
- Add `--threshold` option to `judge_cmd` in `cli.py`
- When `overall_score < threshold`, exit 1 (matching `eval` command's behavior)
- ~10 lines of code change

### Feature: `compare` module + CLI command

**New file: `trajeval/trajeval/compare.py`**
- `ComparisonResult` model: per-metric deltas, overall delta, regression flag
- `compare_reports(baseline: EvalReport, current: EvalReport) -> ComparisonResult`
- Markdown report generation function
- Statistical helpers: delta, direction, regression detection with configurable tolerance

**Modified: `trajeval/trajeval/cli.py`**
- New `compare` subcommand accepting two trace files
- Options: `--format` (table/json/markdown), `--tolerance` (regression threshold, default 0.05)
- Exit code: 0 if no regression, 1 if regression detected

**New file: `trajeval/tests/test_compare.py`**
- Test delta computation, direction, regression detection
- Test tolerance behavior (delta within/outside tolerance)
- Test markdown output format
- Test CLI exit codes

### Design Decisions
- Compare operates on eval results (runs `evaluate()` on both traces), not raw traces
- Tolerance-based regression: score drop > tolerance = regression (default 5%)
- Markdown report includes a summary line suitable for PR comments
