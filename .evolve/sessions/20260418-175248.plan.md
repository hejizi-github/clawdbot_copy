# Session Plan: OpenTelemetry GenAI Trace Ingestion

## Goal
Add OpenTelemetry JSON export ingestion to trajeval, completing the third planned ingestion format (JSON ✓, Clawdbot JSONL ✓, OTLP ✗).

## Direction & Devil's Advocate

**Direction**: Implement `ingest_otlp_json()` in `ingester.py` to read OTel JSON exports and map spans to TraceStep objects.

**Strongest counter-argument**: OpenTelemetry GenAI semantic conventions (`gen_ai.*` attributes) are still evolving — implementing against an unstable spec might require rework later.

**Rebuttal**: The core OTel span structure (trace_id, span_id, attributes, events) is stable. GenAI attributes are additive. Even if specific attribute names change, the structural mapping (spans → TraceSteps) would remain valid. And the JSON export format itself is very mature.

## Changes

1. **`trajeval/ingester.py`** — Add `ingest_otlp_json()`:
   - Parse OTel JSON export format (resourceSpans → scopeSpans → spans)
   - Map span kind + attributes to TraceStep type (llm_call, tool_call, etc.)
   - Extract GenAI attributes: `gen_ai.system`, `gen_ai.request.model`, `gen_ai.usage.*`
   - Handle span hierarchy (parent_span_id) for ordering
   - Compute token usage from `gen_ai.usage.input_tokens`/`output_tokens`

2. **`trajeval/cli.py`** — Add `otlp` to `--input-format` choices

3. **`trajeval/tests/test_ingester.py`** — Add OTel ingestion tests:
   - Basic single-span ingestion
   - Multi-span trace with tool calls
   - GenAI attribute extraction
   - Missing/malformed data handling
   - File not found / empty file errors
