# Session 20260417-111906 Plan

## Goal
Implement SQLite storage module — the foundational persistence layer from the architecture diagram that was never built.

## Why This Is Most Impactful
- SQLite storage is explicitly in the architecture design (proposal Section 3) but missing from the codebase
- Without storage: every command requires explicit file paths, no evaluation history, no auto-baselines
- With storage: enables `trajeval history`, trending, auto-baseline comparison — all post-MVP features depend on this
- Reviewer said to push core features, not do more test maintenance — this is the next logical infrastructure piece

## Approach
1. Create `trajeval/storage.py`:
   - `EvalStore` class wrapping sqlite3
   - Tables: `evaluations` (trace_id, timestamp, overall_score, passed, metrics_json, config_json)
   - Methods: `save_eval()`, `get_eval()`, `list_evals()`, `get_latest()`
   - Default DB path: `~/.trajeval/history.db`

2. Add `trajeval history` CLI command:
   - Lists past evaluation results from the store
   - Table and JSON output formats

3. Add `--store/--no-store` flag to `eval` command:
   - When enabled, persists the EvalReport to SQLite after evaluation

4. Create `tests/test_storage.py`:
   - Test CRUD operations with tmp_path fixture
   - Test listing/filtering
   - Test auto-creation of DB
