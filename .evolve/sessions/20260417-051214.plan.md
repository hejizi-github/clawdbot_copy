# Session 20260417-051214 Plan — Calibration Module (Phase 3 Session 6)

## Goal
Implement the calibration module: human annotation storage, correlation analysis between human and LLM-as-judge scores, and `trajeval annotate` CLI command.

## Why This
- Calibration is the "key differentiator" in the project proposal (Section 3.4)
- Reviewer explicitly recommended it as Priority 2 for this session
- Without calibration, LLM judge scores are ungrounded — no way to know if they correlate with human judgment
- This is also listed in the MVP plan (Session 5) but was deferred for CLI test debt

## Pre-fix
- Rename `test_error_trace_has_lower_scores` → `test_error_trace_parseable` (reviewer noted weak assertion name)

## Scope

### 1. Data models (`trajeval/calibration.py`)
- `HumanAnnotation(BaseModel)`: trace_id, dimension, human_score (0-5), annotator, timestamp
- `CalibrationResult(BaseModel)`: correlation coefficient, p-value, sample size, per-dimension breakdown
- `AnnotationStore`: JSONL-based storage for annotations (append-only, one file per trace or global)

### 2. Correlation analysis
- `compute_correlation(annotations, judge_results)` → CalibrationResult
- Spearman rank correlation (scipy.stats.spearmanr)
- Per-dimension and overall correlation
- Minimum sample size warning (n < 10)

### 3. CLI commands
- `trajeval annotate <trace.json>` — interactive: shows trace, prompts for dimension scores, saves to annotations file
- `trajeval calibrate --annotations <file> --judgments <file>` — computes and displays correlation

### 4. Tests
- Annotation storage: save/load roundtrip
- Correlation: known data → expected Spearman coefficient
- CLI: annotate saves correctly, calibrate outputs valid results
