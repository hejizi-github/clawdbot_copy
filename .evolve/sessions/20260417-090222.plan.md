# Session 31 Plan — Fix 3 review bugs in improve command

## 目标
修复 Session 30 评审发现的 3 个 bug：显示刻度不匹配、CLI 无法 judge-only 分析、metric_summary 键冲突静默覆盖。

## 方案

### 1. 显示刻度不匹配（Priority 1 — 用户可感知 bug）
- **文件**: `trajeval/cli.py:435`
- **问题**: `score_color` 用 `>= 0.7` / `>= 0.5` 判断颜色，对 0-1 确定性指标正确，但 judge 维度 0-5 刻度下 2.0/5（差）也显示绿色
- **方案**: 在 metric_summary 中添加 `scale` 字段（确定性指标 = 1.0，judge = 5.0），`_print_improvement_report` 中归一化后再判断颜色。改动点：
  - `improvement.py` `analyze_results()` 添加 `"scale": 1.0` 到 summary
  - `improvement.py` `analyze_judge_results()` 添加 `"scale": 5.0` 到 summary
  - `cli.py:435` 读取 scale，归一化 `mean_score / scale` 后判断颜色

### 2. CLI 无法 judge-only 分析（Priority 2）
- **文件**: `trajeval/cli.py:371`
- **问题**: `eval_files` 设为 `required=True`，无法仅传 `--judge-files`
- **方案**: 改 `required=False`，在函数体开头加验证 `not eval_files and not judge_files` 时 exit(1)

### 3. metric_summary 键冲突风险（Priority 3 — 防御性）
- **文件**: `trajeval/cli.py:412`
- **问题**: `{**eval_report.metric_summary, **judge_report.metric_summary}` 同名键静默覆盖
- **方案**: 给 judge 维度的 summary key 加 `judge:` namespace 前缀。改动点：
  - `improvement.py` `analyze_judge_results()` 中 `metric_summary[f"judge:{name}"]` 替代 `metric_summary[name]`
  - 确保 finding 的 metric 字段也用同样前缀保持一致

### 4. 修正误导性测试 `test_judge_files_only`
- **文件**: `tests/test_cli.py:511`
- **问题**: 名称暗示 judge-only，但实际传了 eval 文件
- **方案**: 重命名为 `test_judge_files_with_eval`，新增真正的 `test_judge_only_without_eval_files`
