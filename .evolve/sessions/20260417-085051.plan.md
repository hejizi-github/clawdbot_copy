# Session 20260417-085051 Plan

## Objective
Integrate LLM judge results into the improvement analysis pipeline — currently `improvement.py` only handles deterministic `EvalReport`, leaving judge results isolated from the improvement loop.

## Why This Is the Most Impactful Change
1. The review explicitly suggested "LLM judge 结果集成到 improvement analysis" as the next functional step
2. It closes a real gap: users who run `trajeval judge` can't feed results into `trajeval improve`
3. The deterministic metrics module is mature (3 sessions of convergent review); continuing to polish it has near-zero marginal value
4. This makes the system more cohesive — both evaluation paths feed into a single improvement loop

## What to Change

### 1. `improvement.py`
- Add a new function `analyze_judge_results(results: list[JudgeResult]) -> ImprovementReport` that:
  - Aggregates dimension scores across multiple judge evaluations
  - Detects low-scoring dimensions, trends, and high variance
  - Generates dimension-specific recommendations (new `_DIMENSION_ADVICE` dict)
- Add a unified `analyze_all(eval_reports, judge_results) -> ImprovementReport` that merges both
- Keep existing `analyze_results()` unchanged for backward compatibility

### 2. `cli.py` — `improve` command
- Add `--judge-files` option to accept judge result JSON files alongside eval results
- Unified output that shows both metric and judge dimension findings

### 3. Tests
- `test_improvement.py`: Tests for `analyze_judge_results()` covering:
  - Low-scoring dimension detection
  - Trend detection across multiple evaluations
  - High variance detection
  - Empty input handling
  - Mixed analysis (both eval + judge results)
