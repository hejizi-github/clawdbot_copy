# Session Plan: Batch Evaluation Command

## 目标
Add `trajeval batch <directory>` command to evaluate multiple traces at once and produce aggregated statistics — the most natural next feature for CI pipeline usage.

## 方案

### New file: `trajeval/batch.py`
- `batch_evaluate(traces, config)` → `BatchResult` model
- `BatchResult` contains: individual results, aggregate stats (mean scores, fail rate, worst performers per metric)
- Reuses existing `metrics.evaluate()` for each trace

### Modified file: `trajeval/cli.py`
- Add `batch` command: `trajeval batch <directory> [--format table|json|ci] [--input-format auto|json|clawdbot] [--threshold] [--details]`
- Glob-based file discovery (*.json, *.jsonl)
- Exit code 1 if any trace fails threshold

### New file: `tests/test_batch.py`
- Test batch evaluation with multiple traces
- Test aggregation logic (mean, fail rate)
- Test empty directory handling
- Test mixed pass/fail results
- Test CLI integration

## 验证
- `cd trajeval && python -m pytest tests/test_batch.py -v` — all new tests pass
- `cd trajeval && python -m pytest -q` — all 440+ tests still pass
- Manual test: `python -m trajeval batch tests/fixtures/` (if fixtures exist in a dir)
