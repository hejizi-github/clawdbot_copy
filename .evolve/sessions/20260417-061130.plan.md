# Session Plan: Add error_recovery Metric

## 目标
为 trajeval 添加第 5 个确定性评估指标 `error_recovery`，衡量 Agent 从错误中恢复的能力。

## 背景
当前有 4 个指标：step_efficiency, tool_accuracy, loop_detection, token_efficiency。
其中 tool_accuracy 衡量"错误率"，但缺少"错误后是否能恢复"的维度。
评审连续多次建议增加新评估维度和测试覆盖。

## 方案

### 1. metrics.py — 添加 `error_recovery` 函数
- 逻辑：扫描 trace steps，找到所有 error steps
- 对每个 error step，检查后续 N 步内是否有成功操作（非 error 的 tool_call 或 llm_call）
- recovery_rate = recovered_errors / total_errors
- 无 error 时 score=1.0（完美轨迹无需恢复）
- details 包含: total_errors, recovered, unrecovered, recovery_window

### 2. metrics.py — 将 error_recovery 加入 evaluate()
- 在 results 列表中添加 error_recovery 调用
- 自动纳入 pass_threshold 覆写逻辑

### 3. 测试 — test_metrics.py 中添加 error_recovery 测试
- 无 error → score 1.0
- 全部恢复 → 高分
- 部分恢复 → 中等分数
- 无恢复 → score 0.0
- 空 trace → score 1.0
- evaluate() 返回 5 个指标

### 4. test_cli.py — 验证 eval JSON 输出包含新指标

### 5. 新 fixture — recovery_trace.json
- 一个包含 error→recovery 和 error→未恢复 的混合轨迹
