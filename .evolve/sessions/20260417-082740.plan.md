### Part 2: Improvement loop module (core work)
New file: `trajeval/improvement.py` + `tests/test_improvement.py`

Design:
- `analyze_results(results: list[EvalResult]) -> ImprovementReport`
- Identifies patterns across multiple evaluations: which metrics consistently fail, which dimensions score lowest
- Generates actionable recommendations based on failure patterns
- `ImprovementReport` model: findings, recommendations, priority levels
- CLI command: `trajeval improve <results_dir>` — reads multiple eval JSONs, outputs improvement report

Key patterns to detect:
- Repeated loop detection failures → "Agent enters repetitive patterns; consider adding loop-breaking logic"
- Low tool accuracy → "Tool selection needs improvement; review tool descriptions"
- Low step efficiency → "Agent takes too many steps; consider condensing reasoning"
- Low judge scores on specific dimensions → "Focus on improving {dimension}"
- Regression between runs → "Performance dropped on {metric} since {baseline}"
