# Session Plan: format_markdown() details support (Phase 3 Session 17)

## 目标
为 `format_markdown()` 添加 MetricDelta details 展示支持，使 markdown 报告与 JSON 输出信息量对等。

## 背景
Session 16 给 MetricDelta 新增了 baseline_details 和 current_details 字段，JSON 输出已包含这些数据，但 `format_markdown()` 仍只显示分数表格。评审明确建议补上。

## 方案

### 修改文件
1. **`trajeval/compare.py`** — `format_markdown()` 函数
   - 在分数表格之后，为每个有 details 的 metric 添加 collapsible `<details>` section
   - 使用 GitHub-flavored markdown 的折叠语法，保持报告简洁
   - 对 baseline 和 current 的 details 分别渲染为 key-value 列表
   - 如果两侧 details 都为 None，跳过该 metric

2. **`trajeval/tests/test_compare.py`** — 新增 TestFormatMarkdown 测试
   - 测试有 details 时生成 `<details>` 块
   - 测试无 details 时不生成多余内容
   - 测试只有一侧有 details 的场景
   - 测试 details 中的 key-value 被正确渲染

### 对照 checklist（来自经验"新模块必须对齐已有模式"）
- [x] JSON 输出已包含 details — 需要 markdown 对齐
- [x] format_markdown() 当前签名不变（仍接受 ComparisonResult）
- [x] 新渲染逻辑只在 details 存在时才输出，不影响现有输出格式

## 验证
```bash
python -m pytest trajeval/ -q
```
全部通过 + 新增测试覆盖 details 渲染的 4 个场景。
