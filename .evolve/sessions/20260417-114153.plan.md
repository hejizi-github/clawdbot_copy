### OTLP JSON structure (from OTel GenAI semantic conventions)
```
resourceSpans[].scopeSpans[].spans[] → each span maps to a TraceStep:
  - gen_ai.operation.name=chat/text_completion → type="llm_call"
  - name="execute_tool {tool_name}" → type="tool_call"
  - status.code=2 (ERROR) → type="error"
  - duration: (endTimeUnixNano - startTimeUnixNano) / 1_000_000
  - tokens: gen_ai.usage.input_tokens, gen_ai.usage.output_tokens
  - agent: gen_ai.agent.name or resource service.name
  - trace_id: traceId from spans
```

### Mapping details
| OTLP Attribute | AgentTrace/TraceStep Field |
|---|---|
| traceId | trace_id |
| gen_ai.agent.name / service.name | agent_name |
| gen_ai.operation.name | step type (chat→llm_call, execute_tool→tool_call) |
| gen_ai.request.model | step name (for llm_call) |
| gen_ai.tool.name | step name (for tool_call) |
| gen_ai.usage.input_tokens | tokens.prompt |
| gen_ai.usage.output_tokens | tokens.completion |
| startTimeUnixNano/endTimeUnixNano | duration_ms |
| gen_ai.input.messages | step input |
| gen_ai.output.messages | step output |
