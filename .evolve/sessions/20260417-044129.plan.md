# Session Plan: LLM-as-Judge Scorer

## Goal
Implement the LLM-as-judge scoring module — trajeval's core differentiator from pure rule-based eval tools.

## Scope (from proposal Session 3)
- Structured rubric prompts for **task completion** + **reasoning quality**
- Anthropic SDK integration with prompt caching
- JSON output mode for CI
- Mock-based tests (no real API calls in test suite)

## Design

### New file: `trajeval/scorer.py`
- `JudgeConfig(BaseModel)` — model name, dimensions, temperature
- `JudgeDimension(BaseModel)` — name, score (0-5), explanation
- `JudgeResult(BaseModel)` — dimensions list, overall score (normalized 0-1)
- `build_rubric_prompt(trace)` — construct the system+user prompt pair
- `judge(trace, config)` — call Anthropic API, parse structured response
- Prompt caching: system prompt (rubric) gets `cache_control: ephemeral`

### New file: `tests/test_scorer.py`
- Test prompt construction (verify rubric includes trace data)
- Test response parsing (mock API returns structured JSON)
- Test error handling (API failure, malformed response)
- Test score normalization (0-5 rubric → 0-1 normalized)

### Modify: `trajeval/cli.py`
- Add `judge` subcommand: `trajeval judge <trace.json> [--model] [--format]`

### Modify: `trajeval/metrics.py`
- No changes — judge scores are separate from deterministic metrics
