# Plan: PyPI Packaging Readiness + CLI --input-format Tests

## Goal
Make trajeval pip-installable with a working `trajeval` CLI entry point, and close the CLI integration test gap for `--input-format` identified in the last review.

## Rationale
- `[project.scripts]` in pyproject.toml is empty — users can't run `trajeval` after `pip install`
- Last review flagged: "no CLI integration tests covering the new --input-format option"
- These two together make trajeval actually shippable

## Changes

### 1. Fix pyproject.toml packaging
- Add `[project.scripts] trajeval = "trajeval.cli:main"` entry point
- Add `py.typed` marker for typed consumers
- Verify `python -m build` produces a working wheel

### 2. CLI integration tests for --input-format
- Test `--input-format auto` with .jsonl file (should auto-detect clawdbot)
- Test `--input-format auto` with .json file (should auto-detect json)
- Test `--input-format clawdbot` explicit selection
- Test `--input-format json` explicit selection
- Test error case: invalid format with wrong file type

### 3. Verify
- `python -m build` succeeds
- `pip install -e .` works and `trajeval --version` runs
- All 426+ tests pass including new ones
- `ruff check` clean
