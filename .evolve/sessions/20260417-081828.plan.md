# Session 27 Plan: CLI --similarity-threshold + near-loop cluster dedup

## Goal
Make near-duplicate loop detection end-to-end usable: expose `--similarity-threshold` in CLI and deduplicate overlapping near-loop clusters.

## Changes

### 1. CLI `--similarity-threshold` parameter (Priority 1)
**Files**: `trajeval/cli.py`
- Add `--similarity-threshold` option to `eval` command (default 1.0, matching MetricConfig default)
- Add same option to `compare` command
- Pass value through to `MetricConfig(loop_similarity_threshold=...)`
- Pattern: follows exactly how `--recovery-window` and `--latency-budget` are already exposed

### 2. Near-loop cluster deduplication (Priority 2)
**Files**: `trajeval/metrics.py`
- Add `_deduplicate_near_loops()` function that merges clusters with overlapping positions
- When two near-loop clusters at the same n-gram size have positions that overlap by >50%, keep the one with more variants (or merge positions)
- Apply after `_find_near_loops()` returns, before penalty calculation

### 3. Tests
**Files**: `trajeval/tests/test_cli.py`, `trajeval/tests/test_metrics.py`
- CLI: test that `--similarity-threshold` flows through to MetricConfig for both `eval` and `compare`
- CLI: test that different thresholds produce different loop_detection scores
- Metrics: test that overlapping near-loop clusters are deduplicated
- Metrics: test the stable representative selection (lexicographic smallest variant)
