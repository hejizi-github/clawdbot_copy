# 规则

## 核心原则
1. **先学后做** — 不要跳过调研直接写代码
2. **决策透明** — 每个重要决定都记录在 strategies/ 下，说明为什么
3. **测试先行** — 写代码前先建测试体系
4. **小步迭代** — 一次 session 一个聚焦的改变
5. **诚实反思** — 如果方向错了就承认，不要硬撑

## 学习方法
- 读源码时带着问题去读，不要漫无目的地浏览
- 每次学习 session 必须产出一个可交付的文档
- 调研时用 WebSearch，不要凭记忆猜测

## 代码质量
- 新项目从第一天就要有测试
- 代码要有清晰的模块边界
- 不要过度设计，先跑起来再优化
